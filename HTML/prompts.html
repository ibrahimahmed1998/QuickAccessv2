CODING GUIDELINES & BEST PRACTICES
OOP & SOLID Principles
• Use OOP & SOLID Principles in all code implementations
• Single Responsibility: Each class should have one reason to change
• Open/Closed: Open for extension, closed for modification
• Liskov Substitution: Derived classes must be substitutable for base classes
• Interface Segregation: Don't force clients to depend on unused methods
• Dependency Inversion: Depend on abstractions, not concretions
Code Quality & Performance
• Use Top = 1 instead of First() or FirstOrDefault()
• Use Record Codes not GUID (Custom Field preferred)
• Add NoLock within queries: var q = new QueryExpression("EntityName") { NoLock = true; };
• Execute Multiple Transaction instead of for loop updates
• Don't use dynamic or var - Use the matched data type explicitly
• Add Logs and Traces for debugging and monitoring
Communication Style

DYNAMICS 365 / POWER PLATFORM SPECIFIC
Query Best Practices:
✓ Use QueryExpression with NoLock = true
✓ Use Top = 1 for single record retrieval
✓ Filter by Record Code instead of GUID when possible

Bulk Operations:
✓ Use ExecuteMultipleRequest with transactions
✓ Avoid for loops for update/create operations
✓ Handle partial success scenarios

Type Safety:
✓ Use Entity early-bound classes
✓ Avoid 'dynamic' keyword
✓ Avoid 'var' unless type is obvious from right side
✓ Use OptionSetValue, Money, EntityReference types explicitly
CODE EXAMPLES
✅ GOOD - Use Top = 1:
QueryExpression query = new QueryExpression("account")
{
  NoLock = true,
  TopCount = 1
};
EntityCollection results = service.RetrieveMultiple(query);

❌ BAD - Using First():
var accounts = service.RetrieveMultiple(query).Entities.First();

✅ GOOD - ExecuteMultiple:
ExecuteMultipleRequest multipleRequest = new ExecuteMultipleRequest()
{
  Settings = new ExecuteMultipleSettings() { ContinueOnError = true, ReturnResponses = true },
  Requests = new OrganizationRequestCollection()
};

❌ BAD - For Loop Updates:
foreach(Entity entity in entities)
{
  service.Update(entity); // Slow, multiple round trips
}
